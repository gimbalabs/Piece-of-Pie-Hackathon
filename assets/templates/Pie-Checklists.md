
## Builder Pie Final Qualification Checklist

Use this checklist before final submission.

### Product

* [ ] The project is a **user-facing software product**.
* [ ] The product includes a clear payment gate, such as:

  * [ ] subscription pricing
  * [ ] one-time purchase
  * [ ] usage-based pricing
  * [ ] another clear paid access model

### Public Repository

* [ ] The repository is public.
* [ ] The repository will remain public until final payouts are complete.

### Weekly Public Progress

* [ ] Links to all required weekly tweets are collected for the final presentation.

### Public Product Access

* [ ] The product is publicly accessible by the end of the even and the public product link works.
* [ ] The product does not only run locally. It must be 'hosted' somewhere.
* [ ] The deployed product matches the intended platform, such as:

  * [ ] live web app
  * [ ] downloadable app
  * [ ] browser extension
  * [ ] other public deployment (IOS store, Play store, etc.)

### Final Presentation
The final presentation includes

* [ ] a live demo of the product.
* [ ] the official public repository link.
* [ ] links to all required weekly tweets.
* [ ] the deployed public product link.

### Disqualification Check

* [ ] No required weekly tweet is missing.
* [ ] All qualification evidence is public and verifiable.
* [ ] The project can be reviewed through public links without private access.






## Real User Pie Final Qualification Checklist

Use this checklist before final submission.

### Builder Qualification

* [ ] The project qualifies for the Builder Pie or Cardano Pie.

### Paying User

* [ ] The project has secured at least one real paying user.
* [ ] The paying user is not a family member or friend.
* [ ] The payment was made for access to, use of, or purchase of the product.

### Payment Evidence

* [ ] Verifiable payment evidence is available, such as:

  * [ ] blockchain transaction record
  * [ ] invoice marked paid
  * [ ] billing dashboard record
  * [ ] bank transfer record
  * [ ] app store or platform payout record
  * [ ] payment receipt
  * [ ] other clear payment record

* [ ] The evidence clearly shows:

  * [ ] a payment happened
  * [ ] the amount paid
  * [ ] the date of payment
  * [ ] the payment was for the product
  * [ ] enough context to show it came from a real customer

### Customer Acquisition Story

* [ ] A short customer acquisition story is included.
* [ ] The story explains how the paying user was found or acquired.
* [ ] The story shows real-world outreach, validation, or demand.

### Blockchain Payment Check

For blockchain payments:

* [ ] The transaction record is included.
* [ ] Supporting context is included to show the payment was a real customer purchase.
* [ ] The transaction is not just a wallet transfer without customer context.

### Public Verification

* [ ] The payment evidence is shared publicly for verification.
* [ ] The customer acquisition story is shared publicly for verification.
* [ ] All required evidence can be reviewed without private access.




## Cardano Pie Final Qualification Checklist

Use this checklist before final submission.

### Builder Qualification

* [ ] The project satisfies every Builder Pie requirement.


### Cardano Mainnet Functionality

* [ ] The project delivers real functionality on **Cardano mainnet**.


### Cardano Mainnet Evidence

* [ ] Public evidence is available to verify Cardano mainnet functionality, such as:

  * [ ] mainnet transaction links
  * [ ] deployed contract or policy IDs
  * [ ] wallet interaction evidence
  * [ ] public product flows that use Cardano mainnet
  * [ ] repository code that supports the presented Cardano functionality
  * [ ] other clear Cardano mainnet evidence
