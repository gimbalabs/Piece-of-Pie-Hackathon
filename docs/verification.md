# Verification

## Purpose

Verification is how the hackathon confirms that participants met the published requirements.

Verification is based on public evidence and final presentations.

## Builder Verification

To verify Builder qualification, participants must be able to show:

- the official public repository
- visible commit history
- all required weekly tweet links
- a live demo
- a deployed public product link

## Real User Verification

To verify Real User qualification, participants must be able to show:

- proof that the project qualified as a Builder
- proof of at least one paying user
- the customer acquisition story
- public evidence of the above

## Feedback Verification

To verify Feedback qualification, participants must be able to show:

- evidence of live feedback sessions
- recordings for those sessions
- enough information to validate credits earned

## Verification Standard

The standard is verifiable public participation.

If required evidence is missing, incomplete, or not public where required, qualification may be denied.

## Final Presentation Checklist

Builders should be ready to present:

- live demo
- repo link
- all weekly tweet links
- deployed product link
